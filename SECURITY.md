# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly. **Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, send an email to **security@loggielabs.com** with:

- A description of the vulnerability
- Steps to reproduce the issue
- Any relevant logs, screenshots, or proof-of-concept code
- Your assessment of the severity (if possible)

## Scope

This policy covers security issues in the source code, dependencies, build tooling, and published artifacts of this repository. It does not cover third-party services, infrastructure, or issues outside the control of this project.

## What to Expect

- **Acknowledgement:** You will receive an initial response within 72 hours confirming we received your report.
- **Assessment:** We will evaluate the report and determine severity and impact. We may follow up with questions.
- **Resolution:** We aim to provide a fix or mitigation within 90 days of a confirmed vulnerability, depending on complexity.
- **Disclosure:** We will coordinate with you on public disclosure timing. We ask that you do not disclose the vulnerability publicly until a fix has been released.

## Supported Versions

Only the latest released version of this project is actively supported with security updates. Users are encouraged to stay up to date.

| Version | Supported |
| ------- | --------- |
| Latest  | Yes       |
| Older   | No        |

## Credit

We believe in recognizing the contributions of security researchers. With your permission, we will credit you by name (or alias) in the release notes or changelog when a reported vulnerability is fixed.
